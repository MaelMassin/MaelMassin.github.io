AC13.02 | Lire, exécuter, corriger et modifier un programme

Retour à la liste

Ce que j’ai fait

Dans le cadre de mes projets, j'ai analysé des codes sources existants pour en comprendre la logique, lancé des scripts pour vérifier leur comportement et apporté des modifications pour corriger des bugs ou ajouter des fonctionnalités.

Pourquoi je l’ai fait

Cela permet de s'approprier un projet technique, d'assurer la maintenance d'une application et de garantir que le programme répond correctement aux besoins de l'utilisateur final.

Comment je l’ai fait

Méthode : Analyse ligne par ligne, tests unitaires et débogage pas à pas.
Outils : Visual Studio Code, Python IDLE, Debugger Chrome DevTools.
Ressources : Documentations techniques (MDN, Python.org), Stack Overflow.

Mes difficultés

Comprendre des algorithmes complexes écrits par d'autres personnes ou identifier l'origine d'une erreur silencieuse (qui ne bloque pas le code mais fausse les résultats).

Ce que j’en ai appris

J'ai appris à lire du code de manière structurée et j'ai acquis des réflexes de débogage pour isoler rapidement un problème dans un script complexe.

Ce que je ferais autrement

Je mettrais davantage de commentaires dans mon propre code pour faciliter la relecture par autrui et j'utiliserais plus systématiquement des outils de contrôle de version (Git) pour chaque petite modification.
