SAÉ 1.04 : Création d'un site web personnel

Ce projet consiste en la conception, l'administration et l'hébergement d'un portfolio professionnel. Il valide ma capacité à gérer un service web de A à Z.

Compétences mises en œuvre

  • Déploiement & Hébergement : Configuration d'un dépôt GitHub et déploiement via GitHub Pages pour un accès public.
  • Gestion de version : Utilisation des commandes git add, commit et push pour administrer les mises à jour du site.
  • Intégration Web : Adaptation d'un template Bootstrap responsive, respect des recommandations W3C et optimisation des ressources (images, chemins relatifs).

"L'administration ne se limite pas aux serveurs, elle concerne aussi la gestion rigoureuse des sources et la mise à disposition de services fiables pour l'utilisateur."

Apprentissages Critiques

  • Ce que j'ai fait :J'ai mis en œuvre le service d'hébergement de mon portfolio en configurant un déploiement continu via GitHub Pages. J'ai également administré le cycle de vie du site en gérant les versions du code source.
  • Pourquoi je l’ai fait :Pour garantir la confidentialité des visiteurs (chiffrement TLS/SSL) et éviter de pousser par erreur des fichiers inutiles ou sensibles (fichiers de configuration locale, logs) sur un dépôt public.
  • Comment je l’ai fait :
    • Hébergement :J'ai utilisé l'interface de gestion de mon dépôt GitHub. Dans les paramètres "Pages", j'ai sélectionné la branche main pour qu'elle serve de racine au site.
    • Sécurisation :Le protocole HTTPS a été activé automatiquement par GitHub via un certificat SSL Let's Encrypt, garantissant un accès sécurisé sans intervention technique lourde de ma part.
    • Mise à jour :Chaque modification effectuée en local a été poussée via la commande git push, déclenchant automatiquement un "build" et un déploiement par le service GitHub Actions (visible dans l'onglet Deployments de mon dépôt).
  • Mes difficultés :La difficulté n'était pas dans la configuration serveur (puisque GitHub l'automatise), mais dans la synchronisation. J'ai dû m'assurer que l'arborescence de mes fichiers (dossiers images, CSS, JS) en local correspondait exactement à celle du dépôt distant pour éviter les erreurs 404 lors du rendu final.
